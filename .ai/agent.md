# IIHS 에이전트 운영

## 목적
- IIHS TechData의 `Small overlap frontal` 데이터를 누락 없이 수집하는 프로젝트를 계획, 구현, 검증, 기록의 순서로 운영한다.
- 초기 전체 백필과 RSS 기반 증분 수집을 분리해 재현 가능한 파이프라인으로 만든다.

## 운영 원칙
- 범위 우선: 현재 프로젝트와 직접 관련된 파일만 변경한다.
- 최소 변경: 필요한 설정과 스크립트만 추가하고 실험성 부산물은 격리한다.
- 검증 우선: 로그인, 목록 수집, 파일 다운로드, 저장 무결성을 각각 검증한다.
- 추적 가능성: 결정, 가정, 실패, 검증 증거를 `.ai/runlog` 와 `docs/field-notes.md` 에 남긴다.

## 역할 분해
- Research: 사이트 흐름, URL 패턴, 인증 제약, RSS 구조 확인
- Build: 세션 캡처, 인덱서, 다운로드러, 매니페스트 구현
- Validate: 페이지 수, 파일 수, 체크섬, 재실행 안정성 검증
- Record: 작업지시서, 현장 메모, 런로그, 잔여 리스크 정리

## 완료 기준
- 인증 세션을 재사용할 수 있다.
- `driver-side` 와 `passenger-side` 목록을 전체 인덱싱할 수 있다.
- 파일 다운로드가 인증 쿠키와 함께 안정적으로 재현된다.
- 증분 수집 경로와 재조정 경로가 분리되어 있다.
