# 프로젝트 규칙

## 워크플로우
- 모든 작업은 `Plan -> Implement -> Validate -> Record -> Next` 순서를 따른다.
- 상태는 `queued`, `running`, `blocked`, `done`, `failed`만 사용한다.
- 구현 후 검증 없이 완료로 바꾸지 않는다.

## 출력 계약
- 기본 산출물은 Markdown 또는 JSON을 우선한다.
- 모든 응답은 `결론 + 근거 + 다음 액션` 구조를 유지한다.
- 불명확한 부분은 추정으로 숨기지 말고 `Unknown` 또는 `Open Issue`로 남긴다.

## 데이터 및 보안 규칙
- 비밀번호, 쿠키, 토큰은 코드와 문서에 저장하지 않는다.
- 인증 세션 파일은 로컬 재사용만 허용하고 Git에는 올리지 않는다.
- 대용량 원시 다운로드 파일과 자동 생성 리포트는 추적 대상에서 제외한다.

## 검증 규칙
- 로그인 성공 여부는 UI 기준으로 확인한다.
- 목록 수집은 페이지 수와 샘플 행 검증을 포함한다.
- 파일 다운로드는 최소 1개 샘플에 대해 응답 코드, 파일명, 크기를 검증한다.
- 재실행 시 덮어쓰기/중복/부분실패 복구 전략이 없으면 완료로 간주하지 않는다.
